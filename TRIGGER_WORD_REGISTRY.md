# 触发词注册表

本文是 LoraLoom 及其训练任务的触发词唯一登记表。新数据集、Caption、训练配置、采样提示词和推理工作流都应引用这里登记的词，不要临时创造拼音、姓名或缩写。

## 使用规则

1. 每个角色或概念分配一个独立触发词，不跨角色复用。
2. 优先使用 2–4 个罕见 token 组成的 ASCII 字符串，避免自然语言含义。
3. 禁止直接使用姓名、拼音、中文昵称、常见英文单词、职业缩写或品牌名。
4. 不在触发词中附加日期或流水号。数字通常逐位拆分，会增加需要学习的 token 数量。
5. 分配前必须用目标底模的真实 tokenizer 检查，并先观察固定 seed baseline。
6. 一个训练批次内，Caption、`trigger_word`、sample prompts 和推理提示必须完全一致，包括大小写。
7. 已投入训练的词不得删除；废弃后改为“禁用”并保留原因，防止再次使用。

## 编号规则

`zxqv` 可以附加数字，但仅推荐用于彼此独立、不会混合训练且不会同时加载的 LoRA：

| 格式 | Token 数 | 实际拆分 | 建议 |
| --- | ---: | --- | --- |
| `zxqv1`–`zxqv9` | 3 | `zx` + `qv` + 单个数字 | 可用于独立 LoRA 的简短编号 |
| `zxqv10` | 4 | `zx` + `qv` + `1` + `0` | 可用，但数字逐位增加 token |
| `zxqv001` | 5 | `zx` + `qv` + `0` + `0` + `1` | 不推荐零填充编号 |
| `zxqv_1` | 4 | `zx` + `qv` + `_` + `1` | 不推荐，分隔符额外占用 token |
| `zxqv-1` | 4 | `zx` + `qv` + `-` + `1` | 不推荐，分隔符额外占用 token |
| `zxqv260717` | 8 | `zx` + `qv` + 六个数字 token | 禁止把日期当编号 |

编号词共享 `zx` 和 `qv` 两个主要 token。它们可能产生身份串扰，因此：

- 每个角色单独训练、单独加载一个 LoRA 时，可以使用 `zxqv1`、`zxqv2` 等编号。
- 多个角色放进同一训练集、同一 LoRA，或推理时同时加载多个相关 LoRA 时，不要使用同一前缀加编号；应分配 token 集合不重叠的新随机词。
- 编号只是触发词的一部分，不要在 Caption 中混用无编号的 `zxqv`。
- 即使采用编号，仍需为每个完整词运行固定 seed baseline，数字 token 本身也可能带来数量或文字先验。

## 候选池

候选词只表示已完成 tokenizer 检查，尚未完成 baseline 语义测试。正式使用前必须移入“正式登记”并填写用途。

| 候选词 | Token 数 | Token 拆分 | 说明 |
| --- | ---: | --- | --- |
| `qvzx` | 2 | `qv` + `zx` | 与 `zxqv` 使用相同 token，不适合分配给需要与其同时加载的角色 |
| `xqvz` | 3 | `x` + `qv` + `z` | 无已知自然语言含义 |
| `vqxz` | 3 | `v` + `qx` + `z` | 无已知自然语言含义 |
| `qzxv` | 3 | `q` + `zx` + `v` | 无已知自然语言含义 |
| `zvqx` | 3 | `z` + `v` + `qx` | 无已知自然语言含义 |

不要把 token 组成高度重叠的候选分配给需要在同一提示中组合、同一模型中训练或同时加载的多个概念。此时应重新生成候选并检查其 token 集合。

## 分配流程

1. 生成不含自然语言片段的短 ASCII 候选。
2. 使用目标模型 tokenizer 记录 token ID 和拆分结果。
3. 排除完整常见词、姓名、拼音、缩写及数字序列。
4. 用固定 seed 运行纯触发词 baseline，检查是否存在人物、地点、职业、文字或构图先验。
5. 在本表登记用途、底模、拆分和日期。
6. 批量更新 Caption 后断言旧词为 0、新词位于每条 Caption 开头。
7. 导出后再次检查 ZIP 内 Caption、训练 YAML 和 sample prompts 使用同一个词。
